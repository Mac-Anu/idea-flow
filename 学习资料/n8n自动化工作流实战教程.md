# n8n 自动化工作流实战教程：从零搭建 AI 智能体流水线

## 什么是 n8n？
n8n 是一款免费、开源的 **“可视化自动化工作流工具”**。它类似于国外的 Zapier，或者国内的腾讯集简云。
你可以用拖拽节点（Node）的方式，把不同的软件、API 和大模型（如 DeepSeek、OpenAI）串联起来。

**核心理念**：让软件帮你打工。当你设定好“触发条件”后，n8n 会在后台自动按顺序执行一系列任务。

---

## 1. n8n 的极速部署教程

作为极客，我们最推荐使用 **Docker** 来部署 n8n，这样既干净又不会弄乱你的电脑环境。

### 部署命令
打开终端（比如 WSL 的 Ubuntu 命令行），直接运行：

```bash
sudo docker run -it --rm \
  --name ideaflow-n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```

**参数解释**：
- `-p 5678:5678`：把 n8n 内部的 5678 端口映射到你的电脑上。以后在浏览器输入 `http://localhost:5678` 就能访问 n8n 的图形界面。
- `-v ~/.n8n:/home/node/.n8n`：把 n8n 的数据持久化存到你电脑的 `~/.n8n` 目录里，这样就算你关机或者重启 docker，你辛辛苦苦画的流程图也不会丢。

## 2. n8n 的三大核心概念

要精通 n8n，只需要搞懂三个东西：

### 概念一：Trigger（触发器节点）
每一个自动化流水线，都必须有一个“起点”，这个起点就叫 Trigger。
常见的触发器有：
1. **Webhook**：生成一个网址。只要有人访问这个网址（或者给它发数据），流水线就开始跑。（这就是我们目前用的方式，配合浏览器书签，你一点击，它就触发）。
2. **Schedule (定时器)**：类似于闹钟。比如“每天早上 8 点触发一次”。
3. **App Trigger**：比如“当有新邮件发到你的 Gmail 时触发”、“当有人在 Github 给你点 Star 时触发”。

### 概念二：Action Nodes（动作节点）
触发器后面的所有节点，都叫动作节点。它们负责真正“干活”。
比如：
- **HTTP Request 节点**：用来调用任何网上的 API（比如我们用来调用 Jina 抓取网页，或者把数据发给 IdeaFlow 博客系统）。
- **AI 节点 (比如 DeepSeek)**：负责思考、翻译、总结。
- **Code 节点**：如果你觉得现成的节点满足不了需求，你可以直接在里面写 JavaScript 或 Python 脚本来处理数据。

### 概念三：JSON 与数据流转（最核心部分！）
n8n 里的节点就像工厂流水线上的工人。上一个工人处理完的数据，会**自动打包成一段 JSON 格式的数据**，塞到下一个工人的手里。

比如我们抓取网页的流程：
1. `Webhook` 收到书签发来的网址：`{"query": {"url": "https://nextjs.org"}}`
2. 传给 `Jina 节点`。Jina 提取网址，去抓取网页，把抓回来的 Markdown 文本塞在输出的 JSON 里。
3. 传给 `DeepSeek 节点`。DeepSeek 把抓来的文本翻译成中文。
4. 最后传给 `推送到博客` 的 HTTP 节点。

**💡 绝招：Expression（表达式）**
在任何节点的输入框里，你都可以不用写死数据，而是点一下齿轮⚙️，选择 **Add Expression (添加表达式)**。
比如你写 `{{ $json.query.url }}`，就代表**动态获取上一个节点传过来的数据里的 url 字段**。这就是让流水线真正灵活起来的灵魂！

## 3. 我们的“自动抓取与翻译博客”实战复盘

为了让你彻底弄懂，我们复盘一下你现在用的这个强大的工作流是怎么搭建出来的：

### 第一步：起点 (Webhook)
- **节点类型**：`Webhook`
- **作用**：生成一个专门的链接。我们用书签插件模拟了一个请求发给它，附带了我们要抓取的 `url`。
- **重要开关**：记得在右上角把 Test URL 切换成 **Production (生产环境)** 并点击 **Active (激活)**，这样它才能 24 小时在后台默默工作。

### 第二步：抓取正文 (HTTP Request)
- **节点类型**：`HTTP Request`
- **Method**：`GET`
- **URL**：`https://r.jina.ai/{{ $json.query.url }}`
- **作用**：Jina 是一把非常锋利的手术刀。它顺着我们传过来的网址，把乱七八糟的广告、侧边栏全剔除掉，只把核心正文抠出来，并转换成清爽的 Markdown 格式。

### 第三步：AI 翻译官 (DeepSeek AI)
- **节点类型**：`Advanced AI` -> `Chat Model` 或者直接用 `HTTP Request` 调用 DeepSeek 的 API。
- **System Prompt (系统提示词)**：告诉 AI 它的身份。“你是一个顶级翻译官，必须保留所有的 `#标题`、`**加粗**` 和 `![图片](链接)` 格式，只把文字翻译成中文。”
- **输入数据**：把上一步 Jina 吐出来的长长一坨 Markdown 丢给它。它会慢条斯理地吐出翻译好的中文 Markdown。

### 第四步：推送入库 (HTTP Request)
- **节点类型**：`HTTP Request`
- **Method**：`POST`
- **URL**：`http://localhost:3000/api/webhook/n8n/article`
- **Header 鉴权**：我们在这里加了 `Authorization: Bearer <你的暗号>`，防止别人恶意往你的博客里塞垃圾。
- **Body 参数**：
  ```json
  {
      "title": "默认标题（或者你提取的标题）",
      "content": "{{ $json.message.content }}" // 这里填入 DeepSeek 吐出来的翻译结果
  }
  ```
- **作用**：把最终成品狠狠地砸进你的 IdeaFlow 数据库里！

## 4. 给开发者的进阶玩法建议

掌握了上面的基础，你就可以开始在这个基础上“魔改”了：

1. **做一个自动发推机器人**：加一个 `X(Twitter)` 节点，每次你的博客有新文章，自动用 AI 提炼出 100 字精华并加上网址发出去。
2. **做一个飞书/钉钉报警器**：你的 Next.js 系统如果报错了，可以发个 Webhook 给 n8n，n8n 马上推一条消息到你的手机钉钉上。
3. **做一个每日天气与灵感收集器**：把起点换成 `Schedule` (每天早晨 7 点触发)，加上一个 `Weather` 节点查天气，再加上 `OpenAI` 节点让它每天给你写一句励志短句，最后推送到你的邮箱。

**总结**：n8n 就像是编程世界的“乐高玩具”，只要你的想象力足够丰富，你可以用它把整个互联网的 API 都连成你个人的超级流水线！
